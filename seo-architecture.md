## 1. מבנה ה-URL (Routing)

### חוק הברזל
כל עמוד **חייב** לשבת תחת נתיב העמוד שמעליו בהיררכיה. היררכיית ה-URL חייבת לשקף את היררכיית התוכן.

### תבנית הנתיבים (עד 3 רמות)

```
/                                                          → דף הבית (Home)
/<pillar>/                                                 → Level 1 — פילר ראשי (Pillar)
/<pillar>/<sub-pillar>/                                    → Level 2 — תת-פילר (Sub-Pillar)
/<pillar>/<sub-pillar>/<cluster>/                          → Level 3 — קלסטר (Cluster)
/<pillar>/<cluster>/                                       → קלסטר ישיר תחת פילר (כשאין תת-פילר)
```

> **הערה:** קלסטר יכול לשבת ישירות תחת פילר (2 רמות) **או** תחת תת-פילר (3 רמות) — תלוי בעומק התוכן. אסור לחרוג מ-3 רמות תחת דף הבית.

### כללי Slugs
- **אנגלית בלבד**, kebab-case (לדוגמה: `birth-malpractice`, `cancer-diagnosis-delay`)
- **ללא** תאריכים, מספרי גרסה או פרמטרי query במבנה הקאנוני
- כל URL מסתיים ב-trailing slash (`/`) לעקביות
- **לעולם לא** לשנות slug קיים לאחר פרסום — אם חובה, להוסיף 301 Redirect

### דוגמאות תקפות
```
✅ /medical-malpractice/                                             (פילר — L1)
✅ /medical-malpractice/birth/                                       (תת-פילר — L2)
✅ /medical-malpractice/birth/labor-induction-negligence/            (קלסטר — L3)
✅ /medical-malpractice/cancer-diagnosis-delay/                      (קלסטר ישירות תחת פילר — L2)
✅ /car-accidents/whiplash/                                          (קלסטר ישירות תחת פילר — L2)
❌ /articles/article-birth-malpractice.html                          (מבנה שטוח — אסור)
❌ /birth/medical-malpractice/                                       (היררכיה הפוכה — אסור)
❌ /medical-malpractice/birth                                        (ללא trailing slash — אסור)
❌ /medical-malpractice/birth/labor/induction/negligence/             (עומק > 3 — אסור)
```

### קנוניזציה (Canonical)
כל עמוד חייב להכיל `<link rel="canonical" href="...">` המצביע על ה-URL המלא והסופי שלו (כולל דומיין, כולל trailing slash).

## 2. עמודי פילר (Pillars)

עמודי התווך הראשיים של האתר. כל אחד מהם מכסה נושא-על רחב, מקשר לעמודי הקלסטר שתחתיו, ומהווה את נקודת הכניסה המרכזית לאותו נושא ממנועי החיפוש.

| # | שם הפילר | Slug | URL | סטטוס |
|---|-----------|------|-----|--------|
| 1 | רשלנות רפואית | `medical-malpractice` | `/medical-malpractice/` | מתוכנן |
| 2 | נזיקין ונזקי גוף | `personal-injury` | `/personal-injury/` | מתוכנן |
| 3 | תאונות דרכים | `car-accidents` | `/car-accidents/` | מתוכנן |
| 4 | תביעות ביטוח | `insurance-claims` | `/insurance-claims/` | מתוכנן |

### תכולת עמוד פילר (מינימום)
- כותרת H1 ייחודית עם מילת המפתח הראשית
- הקדמה רחבה על הנושא (400–800 מילים)
- **טבלת תוכן / רשת קלסטרים** — רשימת כל עמודי הקלסטר תחת הפילר, עם קישור וכותרת לכל אחד
- קריאה לפעולה (CTA) ליצירת קשר
- Schema.org: `LegalService` + `WebPage`
- Breadcrumbs (ראה סעיף 5)

## 4. חוקי קישורים פנימיים (Internal Linking)

### 🔒 כלל הברזל
> **בכל פעם שנוצר עמוד קלסטר חדש, חובה להוסיף בתחילתו או בסופו קישור פנימי מדויק לעמוד הפילר שמעליו.**

### הנחיות יישום
1. **קישור מקלסטר → פילר (חובה):**
   - מיקום: פסקת פתיחה **או** סעיף סיכום בסוף העמוד
   - טקסט העוגן (Anchor) חייב לכלול את מילת המפתח של הפילר
   - דוגמה: `בעמוד הראשי שלנו בנושא <a href="/medical-malpractice/">רשלנות רפואית</a> תמצאו סקירה מלאה של כל תחומי התביעה.`

2. **קישור מפילר → כל הקלסטרים שלו (חובה):**
   - ברשת ויזואלית (grid) של כרטיסים או ברשימה מסודרת
   - כל קלסטר חייב להופיע בפילר שלו

3. **קישור בין קלסטרים באותו פילר (מומלץ):**
   - רק כאשר רלוונטי תוכנית
   - לא לזרוע קישורים מלאכותיים

4. **קישור בין פילרים (מותר במשורה):**
   - רק כאשר יש קשר תוכני ישיר (למשל: תאונת דרכים שגרמה לרשלנות רפואית)

5. **אסור:**
   - קישור מקלסטר לקלסטר בפילר **אחר** ללא סיבה תוכנית
   - קישורים מוסתרים או בצבע זהה לרקע
   - קישורי `nofollow` פנימיים
   - יותר מ-3 קישורים לאותו יעד באותו עמוד

## 5. פירורי לחם (Breadcrumbs)

### חובה בכל עמוד
כל עמוד באתר (למעט דף הבית) **חייב** לכלול רכיב ניווט פירורי לחם המראה לגולש את מיקומו בהיררכיה.

### תבנית ויזואלית

| סוג עמוד | תצוגה |
|-----------|--------|
| פילר | `בית > רשלנות בלידה` |
| קלסטר | `בית > רשלנות רפואית > תשניק סב-לידתי` |

### דרישות טכניות
1. **HTML:** רכיב `<nav aria-label="פירורי לחם">` עם רשימה מסודרת `<ol>`
2. **RTL:** חץ ההפרדה `›` או `>` יכוון לפי כיוון הכתיבה (RTL — משמאל לימין)
3. **Schema.org:** חובה להוסיף JSON-LD מסוג `BreadcrumbList`:

   ```json
   {
     "@context": "https://schema.org",
     "@type": "BreadcrumbList",
     "itemListElement": [
       { "@type": "ListItem", "position": 1, "name": "בית", "item": "https://tzachilavi101-prog.github.io/tsachi-lavia-law/" },
       { "@type": "ListItem", "position": 2, "name": "רשלנות רפואית", "item": ".../medical-malpractice/" },
       { "@type": "ListItem", "position": 3, "name": "תשניק סב-לידתי" }
     ]
   }
   ```

4. **נגישות:** כל הקישורים עם טקסט עוגן ברור; הפריט האחרון **ללא** קישור (מצביע על העמוד הנוכחי), עם `aria-current="page"`.
5. **מיקום:** מיד אחרי ה-Navbar, לפני ה-H1 של העמוד.

## 6. Sitemap ו-Robots

### sitemap.xml
- חובה לכלול **את כל** עמודי הפילר והקלסטר
- תדירות עדכון (`changefreq`): `monthly` לפילרים, `yearly` לקלסטרים יציבים
- עדיפות (`priority`): דף הבית 1.0, פילרים 0.8, קלסטרים 0.6

### robots.txt
- לאפשר crawl מלא לכל מבנה הפילר/קלסטר
- להפנות ל-sitemap.xml

## 7. כללי מטא-דאטה לכל עמוד

חובה בכל עמוד (פילר וקלסטר כאחד):

- `<title>` — עד 60 תווים, כולל מילת המפתח הראשית
- `<meta name="description">` — 140–160 תווים, ייחודי לעמוד
- `<link rel="canonical">` — URL מלא של העמוד
- Open Graph: `og:title`, `og:description`, `og:url`, `og:type`, `og:locale="he_IL"`
- Schema.org: `LegalService` / `Article` לפי סוג העמוד
- `<html lang="he" dir="rtl">`

### איסור
- אין להשתמש באותו `<title>` או `description` ביותר מעמוד אחד
- אין קאנוניקל חוצה-דומיין (cross-domain) אלא אם מדובר במיגרציה מכוונת

## 8. הדגשת מראי מקום לפסקי דין (Legal Citations)

### למה זה חשוב
מראי מקום לפסיקה הם **אות אמון מקצועי** באתר משפטי. סימון סמנטי נכון מסייע לסורקי מנועי החיפוש (Google, Bing) להבין שמדובר בציטוט משפטי בעל משקל, ותומך ב-E-E-A-T (Expertise, Experience, Authoritativeness, Trustworthiness) — קריטי בתחום YMYL (Your Money Your Life) של משפט.

### 🔒 חוק הברזל
> **כל אזכור של פסק דין, חוק או תקנה חייב להיות עטוף בתגית סמנטית מתאימה (`<strong>`, `<cite>` או `<blockquote>`), ולא להישאר כטקסט חופשי.**

### שלוש רמות של הדגשה — לפי סוג האזכור

#### רמה 1 — אזכור קצר בתוך פסקה (inline)
כשמזכירים פסק דין בשורה אחת בתוך טקסט רציף.

- **תגית:** `<strong>` עטוף ב-`<cite>` (סמנטית מדויקת יותר)
- **דוגמה:**
  ```html
  בפסיקה המנחה <cite><strong>ע"א 2694/90 הסתדרות מדיצינית הדסה נ' גלעד</strong></cite>
  נקבע עקרון אובדן סיכויי ההחלמה.
  ```

#### רמה 2 — ציטוט ישיר מפסק דין (block quote)
כשמצטטים מילולית קטע מהפסיקה.

- **תגית:** `<blockquote>` עם `<cite>` לייחוס המקור
- **חובה:** attribute `cite="URL"` אם קיים קישור רשמי למקור (נבו / ועדת חוקה)
- **דוגמה:**
  ```html
  <blockquote cite="https://...">
    <p>"בתי המשפט יכירו בפיצוי בגין אובדן סיכויי החלמה..."</p>
    <footer>— <cite><strong>ע"א 2694/90 הסתדרות מדיצינית הדסה נ' גלעד</strong></cite></footer>
  </blockquote>
  ```

#### רמה 3 — רשימת פסיקה/סימוכין בסוף העמוד
עבור עמודים אקדמיים/מאמרים עם מספר רב של מקורות.

- **תגית:** `<section>` עם `<h2>מקורות ופסיקה</h2>`, ורשימה `<ol>` בה כל פריט עטוף ב-`<cite>`
- כל פריט: `<cite><strong>מראה מקום</strong></cite>` + תיאור קצר

### דפוסי זיהוי של מראי מקום בעברית
יש להדגיש את **כל** אלה:

| סוג | דפוס | דוגמה |
|-----|------|---------|
| ע"א (ערעור אזרחי) | `ע"א <מספר>/<שנה>` | ע"א 2694/90 |
| ע"פ (ערעור פלילי) | `ע"פ <מספר>/<שנה>` | ע"פ 1234/05 |
| דנ"א | `דנ"א <מספר>/<שנה>` | דנ"א 7/83 |
| בג"ץ | `בג"ץ <מספר>/<שנה>` | בג"ץ 1234/15 |
| רע"א | `רע"א <מספר>/<שנה>` | רע"א 5768/94 |
| ת"א (המחוזי) | `ת"א <מספר>/<שנה>` | ת"א 1234-05-18 |
| חוק | שם החוק + שנה | חוק הנזיקין, תשכ"ח-1968 |
| סעיף | `סעיף <X>` + שם החוק | סעיף 35 לפקודת הנזיקין |

### Schema.org — Structured Data (אופציונלי אך מומלץ)
עבור עמודים המכילים פסיקה משמעותית, להוסיף JSON-LD מסוג `LegalCase` או `Legislation`:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "citation": [
    {
      "@type": "LegalCase",
      "name": "ע\"א 2694/90 הסתדרות מדיצינית הדסה נ' גלעד",
      "inLanguage": "he"
    }
  ]
}
```

### איסורים
- ❌ אין להשאיר אזכור פסק דין כטקסט חופשי ללא תגית סמנטית
- ❌ אין להשתמש ב-`<b>` (לא סמנטי) — תמיד `<strong>` או `<em>`
- ❌ אין להדגיש בצבע/גודל בלבד דרך CSS ללא תגית סמנטית בסיסית
- ❌ אין להדגיש מילים רגילות בטקסט בעזרת `<strong>` — רק אזכורים בעלי משקל

### עיצוב (CSS)
מומלץ להגדיר סטייל ייחודי לציטוטים משפטיים, למשל:

```css
blockquote {
  border-right: 3px solid var(--accent);
  padding: 1rem 1.5rem;
  background: var(--bg-tertiary);
  border-radius: var(--radius-sm);
  font-style: italic;
}
cite strong { color: var(--accent); font-style: normal; }
```

## 9. מיגרציה מהמבנה הנוכחי

**מצב נוכחי:** כל המאמרים יושבים כקבצים שטוחים בשורש (`article-birth-malpractice.html` וכו').

**מצב יעד:** מבנה תיקיות היררכי לפי הטבלאות לעיל.

**לעשות בעת המיגרציה:**
1. יצירת תיקיית הפילר (לדוגמה `medical-malpractice/`)
2. העברת המאמר הקיים לתוך תת-תיקיה חדשה עם `index.html`
3. הוספת 301 Redirect מה-URL הישן ל-URL החדש (דרך `_redirects` אם GitHub Pages, או מטא-refresh כ-fallback)
4. עדכון כל הקישורים הפנימיים ב-`index.html` הראשי ובמאמרים האחרים
5. עדכון `sitemap.xml`
